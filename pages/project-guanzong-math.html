管综数学复习导航 Math Review Navigator 管綜数学復習ナビ

一个基于 VitePress 搭建的管综数学知识网站，把章节目录、知识点模板、搜索与真题整理成适合反复复习的网页导航。 A VitePress-based study site that turns chapter structure, topic templates, search, and past-paper notes into a reusable review navigator for management-exam math. VitePress で構築した管綜数学の学習サイトで、章構成、知識点テンプレート、検索、過去問整理を反復復習しやすいナビゲーションにまとめています。

角色Role役割
Product Design / Front-end Development / Content Architecture
时间Timeline期間

一、把零散知识整理成可复习的网站结构 1. Turning Notes into a Reviewable Site Structure 一、散らばったノートを復習しやすい構造へ

这个项目的出发点不是单纯做一个笔记站，而是把管综数学里分散的章节、题型方法和历年真题解析，整理成一个真正适合高频回看的复习入口。整体基于 VitePress 搭建，目录按算术、应用题、代数、几何、概率等专题展开，打开首页后可以快速进入当前需要复习的模块，而不是在文档文件夹里来回翻找。 The goal was not just to build a note site, but to turn scattered math topics, methods, and past-paper explanations into a structure that actually supports repeated review. Built with VitePress, the site organizes content by arithmetic, word problems, algebra, geometry, probability, and more, so the homepage acts as a fast entry point instead of a raw document folder. このプロジェクトは単なるノートサイトではなく、散在していた数学の知識点、解法、過去問解説を、反復復習に本当に使える構造へ整理することを目的にしています。VitePress を基盤に、算術、文章題、代数、幾何、確率などのテーマごとに展開し、トップから必要な章へすぐ入れる構成にしました。

💡 设计重点：我把它当作“知识导航系统”而不是普通博客来设计，核心是让内容搜得到、看得顺、后续还能持续补充，而不是只追求一次性的页面展示。 💡 Design Focus: I approached it as a knowledge navigation system rather than a blog. The priority was searchability, clear review flow, and long-term maintainability instead of one-off presentation. 💡 設計のポイント：通常のブログではなく知識ナビゲーションとして設計し、検索しやすさ、復習しやすさ、継続更新のしやすさを優先しました。

二、统一内容模板与章节导航 2. Unified Templates and Chapter Navigation 二、統一テンプレートと章ナビゲーション

为了避免知识页越写越乱，我在内容结构上先定了统一模板：每篇知识点尽量围绕“这一节学什么、核心定义、常用公式、典型题型、解题方法、易错点、例题解析、关联知识点、复习提示”展开。这样不仅能让单页信息层级稳定，也方便后续扩展搜索、侧边栏和专题整理。配合左侧按章节展开的导航，整站更像一本可翻阅的复习手册，而不是无序堆积的 markdown 集合。 To keep the content from becoming messy over time, I defined a stable topic template first: what this section covers, core definitions, formulas, common problem types, solving methods, pitfalls, worked examples, related topics, and review notes. That consistency makes each page easier to scan and also supports future search, sidebar structure, and专题 aggregation. ページ構成がばらつかないように、まず内容テンプレートを統一しました。各ページは「この節で学ぶこと、定義、公式、典型問題、解法、注意点、例題、関連知識、復習メモ」を軸に整理しています。これによりページ単体が読みやすくなるだけでなく、検索やサイドバー、特集ページへの拡張もやりやすくなります。

三、内容真源与后续维护流程 3. Source-of-Truth Content and Maintenance Flow 三、コンテンツの真源と更新フロー

项目在结构上刻意把“知识库”和“docs 展示层”分开：知识库负责沉淀原始知识内容，docs 则负责 VitePress 页面、导航配置和最终展示。第一阶段先通过统一命名和手动同步把流程跑通，后续再逐步升级到自动同步脚本。这种做法的好处是，内容可以持续生长，展示层也保留迭代空间，不会因为前端结构调整而打乱原始学习资料。 Structurally, the project separates the knowledge base from the docs presentation layer. The knowledge base stores the source material, while docs handles VitePress pages, navigation, and rendering. The first phase relies on naming conventions and manual sync, with room to evolve into automation later. This keeps the content stable while still allowing the site structure to improve over time. 構造面では、知識庫と docs の表示層を意図的に分離しています。知識庫は原稿の真源として機能し、docs は VitePress のページ、ナビゲーション、表示を担います。まずは命名規則と手動同期で運用を成立させ、後から自動化へ広げられるようにしています。これにより学習内容を安定して蓄積しながら、サイト側の改善も続けやすくなります。
