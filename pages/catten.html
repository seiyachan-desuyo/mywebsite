贸小饭智能推荐系统 MaoXiaoFan Smart Canteen Recommender 贸小饭 スマート食堂レコメンド

“今天吃什么”校园版决策系统：用自然语言对话快速收集位置、赶课状态、口味和预算，在 30 秒内给出可执行的食堂窗口推荐。 A campus-focused "What to Eat Today" decision system: gathers location, urgency, taste, and budget via natural conversation, then returns actionable canteen recommendations in about 30 seconds. キャンパス向け「今日何食べる？」意思決定システム。自然言語の会話で位置・時間余裕・味の好み・予算を収集し、約30秒で実行可能な食堂レコメンドを返します。

项目简介 Project Overview プロジェクト概要

猫大食堂智能推荐系统（贸小饭）是一个基于 Flask + HTML 的对话式推荐 Demo。系统通过接入 DeepSeek API，让用户用最自然的话表达“我在哪、赶不赶、想吃什么、预算多少”，再将这些信息映射到食堂窗口数据中，快速输出具体菜品建议。 MaoXiaoFan is a Flask + HTML conversational recommendation demo. Powered by DeepSeek API, it captures user context such as location, urgency, flavor preference, and budget, then maps these signals to canteen data for specific dish suggestions. 贸小饭は Flask + HTML で構築された会話型レコメンドのデモです。DeepSeek API を活用し、位置・時間余裕・味の好み・予算などを自然な会話で取得し、食堂データにマッピングして具体的な料理を提案します。

核心价值： 不要求用户先想清楚偏好，也能完成点餐决策。就算全程回答“都行”，系统也会按校园热门度给出兜底推荐，解决高频“选择困难”场景。 Core Value: Users do not need perfectly defined preferences to get a decision. Even with "anything is fine" answers, the system can fall back to campus-popular options. コア価値： 好みが明確でなくても意思決定が可能。回答が「どれでも可」中心でも、学内人気データで適切な提案を返します。

6步决策链设计 6-Step Decision Chain 6段階の意思決定チェーン

  • 位置定位： 先锁定最近 1-2 个食堂，减少无效选择范围。 Location First: Narrow to the nearest 1-2 canteens. 位置： 最寄りの食堂 1-2 か所に絞り込みます。
  • 场景状态： 区分赶课快吃 / 慢慢吃 / 都行，决定是否优先快取窗口。 Scenario Status: Fast meal vs. relaxed meal affects queue-speed preference. 状況： 急ぎ/ゆっくり/どちらでもで提供速度の優先度を調整。
  • 形态偏好： 米饭、粉面、面点、油炸或都行，筛选主食形态。 Food Form: Rice, noodles, snacks, fried, or flexible. 形態： ご飯・麺・点心・揚げ物などの主食タイプで絞り込み。
  • 口味感受： 清淡、浓郁、麻辣、酱香、酸甜等口味标签过滤。 Flavor Profile: Light, rich, spicy, savory, sweet/sour tags. 味： あっさり/濃厚/辛い/甘酸っぱい等のタグでフィルタ。
  • 忌口排除： 不吃辣、海鲜、猪肉等约束优先剔除。 Restrictions: Exclude spicy, seafood, pork, etc. 除外条件： 辛味・海鮮・豚肉などを除外。
  • 预算限制： 最后按价格区间完成最终排序与输出。 Budget: Final ranking and recommendation by price bands. 予算： 価格帯で最終候補を並べて出力。

对话体验示例 Conversation Example 会話例

用户： 我在宁远楼，1点半有课，有点赶，想吃偏米饭，辣一点但不要太油，预算 12-15。 User: I am near Ningyuan Building, class at 1:30, in a hurry, prefer rice, spicy but not greasy, budget 12-15. ユーザー： 寧遠楼にいて 1:30 に授業。急ぎ。ご飯系、少し辛め、油っこくない、予算 12-15。

贸小饭： 推荐一食堂二楼黄焖鸡米饭（13元）。步行约6分钟，出餐快，辣口但不油，预算匹配。 MaoXiaoFan: Recommend Braised Chicken Rice (13 RMB) on 2F of Canteen 1. About 6 minutes walk, quick serving, spicy but not greasy, budget fit. 贸小饭： 第一食堂2階の黄焖鸡米饭（13元）を推奨。徒歩約6分、提供が早く、辛めだが重すぎず、予算内です。

系统实现思路 Implementation Approach 実装アプローチ

  • 前后端分离： Flask 提供 API，前端以单页对话交互承载体验。 Separated Architecture: Flask backend API + lightweight chat frontend. 分離構成： Flask API と軽量チャットUIの分離設計。
  • 可扩展数据层： 当前可本地维护菜品表，后续可导入 CSV/RAG 扩展知识库。 Extensible Data Layer: Local dish tables now, CSV/RAG integration later. 拡張可能なデータ層： 現在はローカル管理、将来は CSV/RAG 連携へ。
  • 上下文对话： 多轮会话保留用户历史条件，逐步补全缺失信息。 Contextual Dialogue: Multi-turn memory fills missing decision attributes. 文脈会話： 複数ターンで条件を保持し不足情報を補完。
